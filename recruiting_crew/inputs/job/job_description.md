# Frontend React Developer

## Company context

We are building a B2B SaaS platform used by operations and commercial teams to manage workflows, reporting, and customer-facing dashboards. The product is already in production and the next phase is focused on improving performance, usability, maintainability, and delivery speed across the frontend.

## Role mission

Hire a frontend developer who can take ownership of React features end to end, collaborate closely with product and design, and ship polished user experiences in a modern TypeScript codebase.

## Main responsibilities

- Build new product features in React with TypeScript, from UI implementation to integration with backend APIs.
- Maintain reusable component patterns and keep the frontend architecture consistent across modules.
- Translate Figma designs into responsive, accessible, production-ready interfaces.
- Optimize rendering performance, bundle size, perceived speed, and Core Web Vitals for data-heavy screens.
- Write maintainable tests for components, hooks, and critical user flows.
- Work with product managers, designers, and backend engineers to refine requirements and reduce ambiguity before development.
- Review pull requests, document technical decisions, and help improve engineering standards.
- Debug cross-browser issues and ensure a stable experience on desktop and mobile web.

## Core tech stack

- React 18+
- TypeScript
- Next.js or Vite-based React applications
- Zustand or Redux Toolkit for state management
- React Query / TanStack Query for server state
- REST APIs and basic GraphQL consumption
- Tailwind CSS or CSS Modules
- Jest, React Testing Library, and Playwright
- GitHub, pull request reviews, and CI pipelines

## Must-have requirements

- 4+ years of experience in frontend or full stack development with strong frontend ownership.
- 3+ years of hands-on React experience in production environments.
- Strong TypeScript skills, including typing component props, hooks, API responses, and shared UI utilities.
- Experience building reusable components, design-system-aligned interfaces, and responsive layouts.
- Experience consuming APIs, handling loading/error states, and managing client/server state cleanly.
- Solid understanding of frontend performance, accessibility, semantic HTML, and browser debugging.
- Experience writing automated tests for frontend code.
- Comfortable working in agile product teams and communicating tradeoffs clearly in English.

## Nice-to-have requirements

- Experience with Next.js App Router and server components.
- Experience with Storybook and component documentation.
- Familiarity with data visualization libraries such as Recharts, Nivo, or ECharts.
- Experience migrating legacy frontend codebases to modern React patterns.
- Exposure to analytics instrumentation, feature flags, or experimentation platforms.
- Experience mentoring junior developers or leading frontend initiatives.

## Signals of strong fit

- Has shipped complex SaaS dashboards, admin panels, internal tools, or workflow products.
- Can explain why they chose a state management pattern and when to avoid over-engineering.
- Shows measurable impact in performance, UX quality, delivery speed, or code maintainability.
- Mentions collaboration with design systems, QA, accessibility, and product discovery.
- Demonstrates ownership beyond coding, such as architecture decisions, code reviews, and technical documentation.

## Signals of weaker fit

- Mostly marketing sites or CMS work with limited application complexity.
- Strong backend profile but limited React depth.
- No evidence of testing, TypeScript rigor, or collaboration with product/design.
- Mobile-native experience only, without meaningful web frontend delivery.

## What success looks like in the first 90 days

- Ships at least two production features with good code review feedback and low rework.
- Improves one existing frontend area with measurable gains in maintainability, performance, or UX.
- Contributes reusable UI patterns adopted by the rest of the team.
- Becomes a reliable reviewer for frontend pull requests and participates actively in technical discussions.
